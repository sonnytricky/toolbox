# Contributing

Vielen Dank für dein Interesse, zu diesem Projekt beizutragen!

## Wie du beitragen kannst

Es gibt mehrere Möglichkeiten:

- Bugs melden
- Features vorschlagen
- Dokumentation verbessern
- Code beitragen
- Tests schreiben

## Issue erstellen

Bitte prüfe zuerst, ob bereits ein Issue existiert.

Beim Erstellen eines Issues:

- Nutze möglichst ein vorhandenes Template
- Beschreibe das Problem klar
- Füge Logs oder Screenshots hinzu

## Entwicklungsworkflow

1. Repository forken oder Branch erstellen
2. Änderungen implementieren
3. Tests durchführen
4. Commit erstellen
5. Pull Request öffnen

## Commit Richtlinien

Bitte klare Commit-Nachrichten verwenden.

Beispiel:
- feat: neue Login-Funktion
- fix: Fehler bei Benutzerregistrierung
- docs: README aktualisiert

## Pull Requests

Bitte stelle sicher:

- Code baut ohne Fehler
- Tests laufen erfolgreich
- Dokumentation wurde aktualisiert

Checklist:

- [ ] Code getestet
- [ ] Dokumentation angepasst
- [ ] Keine unnötigen Dateien

## Code Stil

Bitte halte dich an:

- konsistente Formatierung
- verständliche Variablennamen
- kurze Funktionen

## Fragen

Bei Fragen kannst du gerne ein Issue erstellen.
